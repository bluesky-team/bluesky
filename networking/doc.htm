Классы:

Uri, Path - классы для удобной работу с URI/путями

Context - класс содержит общие настройки/объекты. Создан главным образом для того чтобы не плодить синглтонов

ResourceManager - умеет обрабатывать сообщения. В ответ на Request выдает Response.
NameService - знает какому каталогу какой ResourceManager соответствует. Т.е. даете ей Path и получаете ResourceManager и путь относительно того каталога за который ResourceManager отвечает.

Connection/LocalConnection/HttpConnection.
Connection отправляет Request по указанному URI и возвращает Response.
Если URI имеет вид "local:///path/", то создается LocalConnection. LocalConnection узнает у NameService ResourceManager и отправляет ему Request напрямую.
HttpConnection создается если URI имеет вид "http://server/path". HttpConnection соединяется с сервером указанном в URI. На серверной стороне соединение принимает HttpServer. HttpServer передает Request ResourceManager'у.

Tree - древовидный контайнер объектов. Само Tree объектов не содержит, а содержит только точки монтирования. Точка монтирования - это пара (путь, URI). По указанному URI должен находиться TreeResourceManager или другой ResourceManager предоставляющий аналогичную семантику.

TreeNode/TreeObjectPtr - смарт-поинтеры на объект которые возвращаемые Tree.
TreeNodeLocker - объект для RAIIной блокировки.

TreeResourceManager - реализация ResourceManager'а рассчитанная на работу с Tree. TreeResourceManager содержит TreeContainer который непосредственно содержит объекты дерева.

SessionContext - содержит данные относящиеся к одному сетевому соединению. На данные момент содержит в себе только блокировки. SessionContext - уничтожается вместе с закрытием соединения, обеспечивая тем самым снятие неснятых блокировок.
